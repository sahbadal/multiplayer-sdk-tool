Complete architecture — one page

Unity Photon Multiplayer SDK

A simplification layer over Photon Fusion + Photon Voice for Unity. Developer fills config through one Setup Window, calls a handful of MultiplayerSDK.* methods — everything else (connection, spawning, matchmaking retries, RPC plumbing, voice lifecycle) is handled internally.

Out of scope: auth, inventory, friends/parties, skill-based backend matchmaking, lobby UI, player visuals, game rules, VR head/hand sync, text chat, custom backend allocation.

Production rules

hard constraints on the implementation, all 20

01Explicit serialized references only, never scene-search
02No GameObject.Find, tags, or naming conventions at runtime
03Resources.Load is never the primary loading mechanism
04Tags are optional, never required
05Folder conventions help, never mandatory
06ScriptableObjects are the single config source of truth
07Every setting has exactly one owner
08Setup Window edits assets, is never a second storage layer
09Never modifies a developer prefab without confirmation
10Safe auto-fixes always separated from destructive changes
11Generated files never overwrite developer-owned assets
12Only one MultiplayerBootstrap per active scene
13Voice remains completely optional
14RPC / event identifiers stay unique and stable
15Matchmaking cancellation is safe from every state
16Common Photon boilerplate is hidden by default
17Advanced Photon APIs stay reachable
18SDK manages networking only — never UI or gameplay
19Config assets are reusable across scenes
20Runtime modules never write to config assets
